## 一、总体审核结论

用正式评阅语言说明：

- 是否达到提交标准。
- 主要风险集中在哪些章节。
- 哪些问题属于必须修改。
- 哪些问题属于建议完善。

禁止写成方法论说明。

## 三、逐处批注式审核意见

每条意见使用固定格式：

### 批注 XXX-000：简短标题

- **追溯编号**：ISSUE-000 / MATRIX-000 / OBJ-000 / FACT-000 / REF-000。
- **定位**：PDF 物理页 pX；论文页码/章节；对象类型。
- **原文摘录**：摘录必要片段，不做大段复制。
- **批注意见**：用评阅语言直接说明该处存在什么问题。
- **审核依据**：说明依据来自自检清单、问题汇总、论文前后文、PDF原图、外部核验或学术规范。
- **修改要求**：给出作者可执行的具体修改动作。
- **优先级**：必须修改 / 建议修改 / 格式修正。
- **确定性**：确认问题 / 疑似问题 / 需作者补充材料 / 无法判断。

## 六、建议修改顺序

按作者修改工作流排序，不按审稿发现顺序排序。

## 七、复核清单

列出作者修改后必须逐项复核的事项。

## 八、覆盖与剩余风险

必须说明：

- 已检查的页码、章节、图表、公式、参考文献和外部事实范围。
- 已检查的方法前提、可验证声明、内部数字一致性和核心计算结果范围。
- 已回看 PDF 原图或页面渲染的高风险位置。
- 因解析质量、材料缺失、内部数据不可得或外部来源不足而无法判断的项目。
- 这些剩余风险对论文提交或后续复核的影响。

禁止把未检查或无法判断的事项写成“未发现问题”。
